ERROR GUIDE

Error Format:
error: [message] in line [#]
> line preview
stack trace
[space] to exit
Example Error:
error: invalid token in line 6
expected {num}, got {str}
> add("apple", 2 => result)
stack trace:
/main
[space] to exit
Miscellaneous:
program finished without errors
No errors were found during execution.
system exit
console(exit) was ran.
Syntax Errors:
no termination statement found
The interpreter couldn't find a termination statement to signal that the code ended.
uneven amount of braces
The amount of opening braces is not equal to the amount of closing braces.
invalid file
A file that doesn't exist was trying to be imported.
Runtime Errors:
parsing overflow
The interpreter's overflow counter is greater than 200. This happens if the interpreter is searching for a specific token but enters an infinite loop due to the token not existing. This error is thrown as a last resort if the command is recognized but malformed. (For example, not enough parameters were present in the command)
missing parentheses
A command doesn't contain the same amount of opening and closing parentheses.
missing opening brace / missing closing brace
You forgot to include a opening or closing brace in an if statement or loop block. This acts as additional protection from crashes if you have an even but malformed amount of braces.
missing arrow assignment
A command that requires arrow assignment doesn't contain proper arrow assignment syntax.
undefined variable
A variable that doesn't exist was referenced.
invalid token
expected {token}, got {token}
The interpreter received an unexpected token in a command.
invalid save slot
You tried to save to or load from an invalid slot.
namespacing collision
You directly tried to edit a scoped variable, such as a function's parameter or a loop counter. This protects scoped variables from being modified outside their intended context.
invalid syntax
A command couldn't be recognized.
can't nest file calls
You tried to call a function from another file which calls its own function. Calls from multiple files have a stack depth limit of 1.
invalid variable nesting level
You tried to nest a variable reference more than two levels deep, such as @@@apple.
lists are different lengths
You tried to perform a list operation on a non-list command (add/sub/mul/div/mod/sqrt/random/concat) but the input lists were not the same length.
Reading Stack Traces:
stack trace:
/main (This is where your code starts)
> /some_function [9] (Jump to '/some_function' in line 9)
 < run(/some_function) [3] (Return to 'run(/some_function)' in line 3)
If the line number shows [module] instead, that means that your code jumped to a function outside the main file.
