COMMANDS LIST

Execution Rules:
1. Direct+ programs start at line 1 and execute top to bottom, unless told to jump somewhere else using the run() system. (More on that below)
2. Termination statements immediately stop program execution (Used as a program stop marker)
Token Guide:
Common Data Types:
{num} = Number. (Including floats and integers)
{str} = String. (Wrap this in quotes!)
{bool} = Boolean value (either true or false)
{any} = Any common data type is accepted.
Special data types:
{list} = A list. Can be a literal list or a variable referencing a list.
{table} = Table name.
{ref} = run() reference.
{slot} = Save/load slot. (1-100 per directory)
{file} = A name of a Direct+ file (without the extension).
Direct+ is one-indexed, and 0 is reserved for special case operations and is not a valid element index. (Like in lists or as a failure method)
Miscellaneous:
;
Terminates code, can be placed anywhere in your program, even multiple times. This is mandatory and your program can't run without it. For the termination statement to register, it should be placed as the first character in a new line of code.
$ comment
Ignored by the interpreter. Useful for documenting code.
Input and Output:
paste(mode)
Interacts with the console. Different modes serve different purposes, as shown below:
paste(text"{str}")
Prints {str} onto the console.
paste(input"{str}" => destination)
Prompts the user with {str} and stores the response.
color({num}, {num}, {num})
Changes the color to hue (0-200), brightness (-100 = black, 100 = white), and transparency (0-100, which acts as saturation since it's not natively supported) respectively. The color will apply to the text until it is changed again. Use 0, 100, 0 to revert back to the standard text.
Variables:
var(name = {any})
Creates or updates a variable with the value {any}. You can also define a variable without a value to set it to an empty value. ("") Snake case recommended.
@variable
References the value of a variable. This can be a substitute for a value in any command as long as the value is the same as the expected type. Use two @'s to double dereference a variable. (See Advanced Topics for more information)
Control Flow:
if({any} ==/!=/>/!>/</!< {any}) <<
>>
Compares two values and runs the code inside if it's true.
else() <<
>>
Runs the code inside if the preceding if statement is false. Else statements must be placed immediately after an if statement or else an else block has no parent error will occur.
loop({num}) <<
>>
Loops the code {num} times. If {num} is 0, the code loops infinitely. Since the interpreter updates every newline and code can be stopped anytime, infinite loops aren't a safety hazard. Nesting is fully supported with all control flow elements. Use loop({num}, {name}) to automatically create a namespaced counter variable in the loop. (See Advanced Topics for more information)
break()
Exits out of a loop early.
run({ref})
Reroutes the interpreter to the specified {ref} (use /reference_name to select a reference). Use /file:function to run a function in another file. The run() system is stack-based, meaning run() chaining is supported along with nesting with all control flow elements. Labels themselves shouldn't be nested. (Nothing really will happen, but it's generally discouraged since that approach just doesn't make sense) These calls can also contain parameters. (See Advanced Topics for more information)
/label
A direct reference to parse from, can be named anything. It can also contain parameters. (See Advanced Topics for more information)
/return
Returns the parser to the last jump, useful for calling functions multiple times. It can also return values. (See Advanced Topics for more information)
Number Operations:
add/sub/mul/div({num}, {num} => destination)
Performs an arithmetic operation on the two values.
sqrt({num} => destination)
Returns the square root of the given value.
mod({num}, {num} => destination)
Returns the remainder of the two values.
round({num} => destination)
Returns the rounded value.
random({num}, {num} => destination)
Returns a random number between the two values. (Inclusive)
String Operations:
concat({str}, {str} => destination)
Returns the two values joined together.
letter({num}, {str} => destination)
Returns the specified index {num} of {str}.
length({str} or {list} => destination)
Returns the length of {str}. If the value is a name of a list, it returns the number of items in that list.
contains({str}, {str} => destination)
Checks if the first string has the second one in it. If so, true is returned. If not, false is returned. If the first string is the name of a list, then it will check if the list contains the second string.
slice({str}, {num}, {num} => destination)
Slices {str} starting at the first number and ending at the last. (Inclusive)
Lists:
var(list_name = {add, some, items, here})
Creates a list. There can be multiple different types in a list. Since lists are just variables, you can turn them into a variable and vice versa, just by changing its value. Nested lists are not supported.
enter({list}, {num}, {any})
Adds {any} at index {num} in {list}. If {num} is 0, then {any} is added to the end of {list}.
delete({list}, {num})
Deletes item {num} of {list}. If {num} is 0, then the whole list is deleted.
replace({list}, {num}, {any})
Replaces item {num} of {list} with {any}.
item({list}, {num} => destination)
Returns the {num}'th item in {list}. Returns an empty string ("") if the index is invalid.
numof({list}, {str} => destination)
Returns the index of the first instance of {str} in {list}. Returns 0 if the item is not found.
Files:
use({file})
Imports the variables from another file and allows you to call functions between the main file and the file you just imported. You can only nest calls between files by 1. Extensions are imported and referenced the same way. (See Extensions for more information)
Persistent Storage:
storage({str})
References a storage directory for you to save and load to. (Creates a new one called {str} if one doesn't exist) Running storage("clear") clears all directories and cannot be undone. Directories can be swapped using the same command but with a different directory.
save({slot}, {any})
Stores {any} in {slot} in the chosen directory. There are 100 slots in each directory. These slots persist until storage(clear) is run. They are overridden with a new value if save() is called with the same slot. If you enter an invalid slot, an Invalid Save Slot error is thrown.
load({slot} => destination)
Loads the current value of {slot} inside the chosen directory.
